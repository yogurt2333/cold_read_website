Problem

普通 AI 分析只给答案，ColdRead 给证据链。

当 AI Agent（智能体）给出预测市场建议时，用户很难知道它看了什么证据、 哪一步出错、是否修复，以及最终材料有没有被篡改。ColdRead 把这些过程 变成 Agent Run Trace（智能体运行轨迹）、Trace Hash（轨迹哈希）和链上 Audit Anchor（审计锚点）。

Workflow

GLM-5.1 驱动的 6 步可审计流程

01

Plan task

GLM-5.1 拆解盘口审计任务，制定多步骤计划。

02

Fetch live market

读取真实 Polymarket 盘口材料，并保留来源链接。

03

Draft trace

生成第一版 Agent Run Trace，记录行动、工具和观察。

04

Validate failed

校验器发现 externalRiskFlags 缺失，拒绝进入审计载荷。

05

Self repair

Agent 补齐风险字段，并将建议降级为 HOLD。

06

Prepare anchor

计算 Trace Hash，并准备 Sepolia calldata 审计锚点。

Boundary

MVP 做审计，不做自动交易。

ColdRead 当前不执行真实 Polymarket 交易，也没有自动操作 Cobo（钱包）。 V2 会加入 Investment Plan（投资计划）和 Cobo Wallet Executor（Cobo 钱包执行器）， 在用户设置的风险预算和人工确认边界内执行小额操作。
